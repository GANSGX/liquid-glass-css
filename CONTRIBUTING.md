# Contributing

Thanks for contributing to `liquid-glass-css`.

## Ground Rules

- Keep styles minimal and reusable.
- Preserve dark-first readability and contrast.
- Avoid adding framework-specific dependencies.
- Use clear commit messages (Conventional Commits preferred).

## Local Setup

```bash
git clone https://github.com/GANSGX/liquid-glass-css.git
cd liquid-glass-css
```

Open examples directly:

```bash
open ./examples/variants.html
```

or run a local server:

```bash
npx serve .
```

## Pull Request Process

1. Create a feature branch from `main`.
2. Keep changes scoped and explain intent in PR description.
3. Update `README.md`/`README.ru.md` when public behavior changes.
4. Add or refresh screenshots if visuals changed.
5. Ensure example pages still render correctly.

## Style Notes

- Prefer compact, readable CSS.
- Reuse existing utility classes when possible.
- Keep naming simple and explicit.
